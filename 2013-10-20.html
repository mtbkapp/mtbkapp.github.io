    For the past year or so I've been working frustrated at my full time job by ColdFusion (the Railo flavor). When I first started working with it I couldn't articulate what was wrong with it. Now, I have few reasons:
    • The language is best described as a crippled JavaScript, script and regular CFML
    • Can't ahead of time compile code without running the entire Railo platform
    • CFML Lists
    • Only data structure literals are structs (more like maps) and arrays.
    • There is no dependency management. I don't think it would have been hard for the designers of Railo or Adobe ColdFusion to provide a way to package ColdFusion code into a jar. Integration with Maven would also make me smile.
    • Minimal Java Integration, I didn't think this was a problem until I learned about Clojure's java integration which is phenomenal. I especially like that with Protocols a Java type can participate in Clojure's abstractions.
    • CFML doesn't help build your web application. In the past I used the OpenACS platform to build web applications. It had the right tools user management, authentication, permissions, dependency management, forced separation of templating and logic, testing framework, and more. This is really it. ColdFusion doesn't give me sharp tools. It could reuse some great Java libraries but to do that I would want better Java interop and the ability to use it with Maven.

    However, there is "no silver bullet". There are trade offs and a continuum of expressiveness that is often subjective. ColdFusion is very easy to get up and running and has been the base of many applications. I just think there are better tools available.
